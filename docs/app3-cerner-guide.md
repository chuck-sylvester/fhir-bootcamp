# App3 - SMART Practitioner App on Cerner

## Requirements

Build a practitioner-facing web application that integrates with the Cerner SMART on FHIR Sandbox:

1. Initiate the launch automatically when opened
2. Display a Patient Banner (or not) depending on what the EHR instructs via token context.
3. List all of the current patient’s vital signs.
4. Allow the user to create new vital sign entries.

## EHR Launch

### Background Info

Read the SMART EHR Launch Flow specification  
[SMART EHR Launch](https://build.fhir.org/ig/HL7/smart-app-launch/app-launch.html#launch-app-ehr-launch)  

Register a Practitioner App on Cerner’s Sandbox
[Cerner Code Sandbox](https://code-console.cerner.com/)  

Two Ways to Render an App from Cerner, once the user is logged in:  
 - iFrame
 - Popup

Example of a full **launch** string that the user's browser may navigate to:
```bash
https://yourapp.com/launch.html?iss=https://fhirendpoint.com&launch=abc1234
```

Another example from the FHIR spec with URL encoding:
```bash
Location: https://app/launch?iss=https%3A%2F%2Fehr%2Ffhir&launch=xyz123
```

 Three key components to know about:  

  - Launch URL (the URL that gets called when my app is launched)
  - `iss` parameter (issuer - base URL of the FHIR server)
  - `launch` parameter (a token generated by the EHR to identify which context the app was launched in and to prevent unnecessary repeated logins)

### Register a Practitioner App on Cerner Sandbox

Website: [https://code-console.cerner.com/](https://code-console.cerner.com/)

**Steps to Create a New Application**

1. Click the `+ New Application` button
2. Complete the Basic Information fields
   - Application Name: FHIR Bootcamp Practitioner App
   - Application Owner: C. P. Sylvester
   - Application Type: Practitioner
   - Type of Access: Offline
   - Application Privacy: Public
   - SMART Version: SMART v2
   - Intended Users: Individual/Caregiver
   - Intended Purposes: Clinical Tools
3. Complete the URIs Information fields
   - Redirect URI: http://localhost:8000
   - SMART Launch URI (optional): http://localhost:8000
   - Default FHIR Version: R4
4. Click `Next` button
5. Complete the Product Selection fields
   - Select a Product Family: Millennium
   - Select Products: Oracle Health FHIR APIs for Millennium: FHIR R4, All
6. Click `Next` button
7. Select User Product APIs: Observation (rscu), Patient (rscu)
8. Select Patient Product APIs: <none selected>
9. Click `Next` button
10. Terms of Use: I agree to the terms of use

This resulted in the creation of a new app, with the following values:
```text

   FHIR® Version: r4
  Application ID: 184da45c-0904-4497-ba4f-5adb9fc266d0
       Client ID: 8f2811bb-ce9c-4657-b660-5232190f0159
```
